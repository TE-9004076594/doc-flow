## Why

企业正式文档生产仍高度依赖 Word/Excel 手动处理，存在模板分散混乱、字段遗漏、格式错乱、审核无留痕、业务数据脱节等痛点。虽然市场上已有通用办公工具和 AI 写作助手，但缺乏能直接生成"可交付正式文档"的自动化平台。本项目旨在构建一套以**模板驱动 + 规则引擎 + AI 装配 + 批量生成**为核心的企业正式文档自动化平台 MVP，快速验证核心闭环。

## What Changes

- **新增模板治理中心**：支持 Word 模板上传、解析、版本管理、分类标签
- **新增文档规则引擎**：支持变量管理、条件块、循环块、基础校验规则
- **新增文档预览与在线编辑**：支持分页预览、变量可视化编辑、轻量文本修改
- **新增文档生成能力**：支持单份生成和基础的批量生成（Excel/CSV 导入）
- **新增文档导出**：支持导出 Word 和 PDF 格式
- **新增基础权限控制**：基于角色（管理员、模板管理员、普通用户）的权限体系
- **新增文档审核流（V1 预留接口）**：MVP 阶段仅建立状态模型

## Capabilities

### New Capabilities
- `template-management`: Word 模板上传、结构解析、分类标签、版本管理（版本号、更新说明、版本对比、回滚）
- `template-variables`: 模板变量管理，含文本/数字/日期/枚举/布尔/对象/列表多类型支持
- `template-logic-blocks`: 条件块（if/else）和循环块（for-each）支持，含嵌套规则
- `rule-validation`: 必填校验、数据格式校验、枚举值校验、文本长度校验、生成前后校验
- `document-preview`: 正式文档分页预览，含页眉页脚、页码、缩放跳页
- `online-editing`: 字段面板可视化编辑与实时刷新、轻量段落修改与文本替换、撤销/恢复
- `document-export`: 导出 Word（.docx）和 PDF 格式，保留模板原始样式
- `document-generation`: 单份手工生成 + 批量导入生成（Excel/CSV 字段映射），异步任务基础支持
- `permission-control`: 基于角色的权限管理（系统管理员、模板管理员、普通用户），模板级查看/编辑/发布权限

## Impact

- **新增代码模块**：8 个核心业务模块，每个模块包含前后端代码
- **新增 API 端点**：模板 CRUD、变量管理、文档生成、导出下载等 REST API
- **依赖关系**：需要 Word 模板解析引擎（如 docx.js/python-docx）、PDF 转换服务
- **数据库**：需新增模板、变量、文档、任务、权限等数据模型
- **基础设施**：需文件存储（模板文件、生成文档）、异步任务队列（批量生成）
