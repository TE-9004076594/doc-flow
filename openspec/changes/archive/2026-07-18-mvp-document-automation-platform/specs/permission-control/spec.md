## ADDED Requirements

### Requirement: System supports role-based access control
The system SHALL implement RBAC with three roles: system administrator, template administrator, and regular user.

#### Scenario: Define system administrator role
- **WHEN** user is assigned the "系统管理员" role
- **THEN** they SHALL have full access to all system features including user management, template management, and all documents

#### Scenario: Define template administrator role
- **WHEN** user is assigned the "模板管理员" role
- **THEN** they SHALL be able to upload, edit, publish, and disable templates
- **AND** SHALL NOT have access to system user management

#### Scenario: Define regular user role
- **WHEN** user is assigned the "普通用户" role
- **THEN** they SHALL be able to view available templates, generate documents, and manage their own generated documents
- **AND** SHALL NOT be able to create or modify templates

### Requirement: System controls template-level permissions
The system SHALL enforce permissions at the template level based on user identity and role.

#### Scenario: View permission
- **WHEN** a non-admin user attempts to view a template they don't have access to
- **THEN** the system SHALL deny access and not show the template in the listing

#### Scenario: Edit permission
- **WHEN** a template attempts to edit a template without edit permission
- **THEN** the system SHALL display an "操作无权限" error message

#### Scenario: Template isolation by organization
- **WHEN** templates are assigned to an organization/team
- **THEN** only members of that organization SHALL be able to access those templates

### Requirement: System supports basic user management
The system SHALL allow system administrators to create, disable, and manage user accounts.

#### Scenario: Create user account
- **WHEN** system admin fills in user information (name, email, role) and clicks "创建用户"
- **THEN** the system SHALL create the user account and send an invitation email

#### Scenario: Disable user account
- **WHEN** system admin disables an existing user account
- **THEN** the user SHALL no longer be able to log in or access the system
- **AND** existing documents generated by the user SHALL remain accessible to permitted viewers

#### Scenario: Assign role to user
- **WHEN** system admin changes a user's role
- **THEN** the user's permissions SHALL be updated immediately upon next action
